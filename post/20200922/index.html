安装

Gentoo的安装和Arch其实没有太大区别，照着Wiki走基本没有太大问题。

由于破船本身已经装了一个Arch了，为了方便查看wiki我选择直接在Arch上把Gentoo安装到另一块ssd上面。

磁盘

破船装了两块ssd，所以我准备把Gentoo安装另一块未使用的ssd(/dev/nvme0n1)上

$ sudo fdisk /dev/nvme0n1

创建三个分区，如下：

Device             Start        End   Sectors  Size Type
/dev/nvme0n1p1      2048    1050623   1048576  512M EFI System
/dev/nvme0n1p2   1050624  944769023 943718400  450G Linux filesystem
/dev/nvme0n1p3 944769024 1000215182  55446159 26.4G Linux swap

选择文件系统

$ sudo mkfs.fat -F 32 /dev/nvme0n1p1
$ sudo mkfs.btrfs /dev/nvme0n1p2
$ sudo mkswap /dev/nvme0n1p3
$ sudo swapon /dev/nvme0n1p3

然后挂载到Arch上

$ sudo mkdir /mnt/gentoo
$ sudo mount /dev/nvme0n1p2 /mnt/gentoo

stage3

磁盘准备好了之后就是安装stage3

stage包有multilib和no-multilib两种，后者是纯64位，按喜好选择，我选择了no-multilib。

下载stage3

stage3包在各个镜像站的gentoo/releases/amd64/autobuilds/下可以找到，由于我想使用systemd而不是默认的OpenRC，所以选择了这个stage3包

current-stage3-amd64-systemd/stage3-amd64-systemd-20200920T214503Z.tar.xz

把stage3包下载到/mnt/gentoo目录下

解压stage3

$ cd /mnt/gentoo/
(/mnt/gentoo)$ sudo tar vxpf stage3-amd64-systemd-20200920T214503Z.tar.xz --xattrs-include='*.*' --numeric-owner

配置编译选项

$ sudo "echo MAKEOPTS=\"-j9\"" >> /mnt/gentoo/etc/portage/make.conf

安装基本系统

配置源

  • 源代码源

在 /etc/portage/make.conf 中添加或修改:

GENTOO_MIRRORS="https://mirrors.ustc.edu.cn/gentoo/"
  • ebulid
$ sudo mkdir /mnt/gentoo/etc/portage/repos.conf

在 /etc/portage/repos.conf/gentoo.conf 中添加或修改:

[DEFAULT]
main-repo = gentoo

[gentoo]
location = /usr/portage
sync-type = rsync
sync-uri = rsync://rsync.mirrors.ustc.edu.cn/gentoo-portage/
auto-sync = yes

复制DNS信息

$ sudo cp --dereference /etc/resolv.conf /mnt/gentoo/etc/

挂载必要的文件系统

$ sudo mount --types proc /proc /mnt/gentoo/proc
$ sudo mount --rbind /sys /mnt/gentoo/sys
$ sudo mount --make-rslave /mnt/gentoo/sys
$ sudo mount --rbind /dev /mnt/gentoo/dev
$ sudo mount --make-rslave /mnt/gentoo/dev

Chroot

$ sudo chroot /mnt/gentoo /bin/bash
root #source /etc/profile
root #export PS1="(chroot) ${PS1}"

挂载boot分区

# mount /dev/nvme0n1p1 /boot

配置Portage

# emerge-webrsync
# eselect news list
# eselect news read
# eselect profile list
# eselect profile set default/linux/amd64/17.1/systemd
# emerge --ask --verbose --update --deep --newuse @world

配置USE

在 /etc/portage/make.conf 中添加或修改:

USE="X alsa pulseaudio udev dbus"                                                          
L10N="en-US zh-CN en zh"

时区语言

  • 选择时区
# echo "Asia/Shanghai" > /etc/timezone
# emerge --config sys-libs/timezone-data
  • 配置地区

编辑 /etc/locale.gen ,添加:

en_US.UTF-8 UTF-8
zh_CN.UTF-8 UTF-8

# locale-gen
# eselect locale list
# eselect locale set *

重新加载环境:

# env-update && source /etc/profile && export PS1="(chroot) ${PS1}"

配置内核

Gentoo官方内核Stable版一般都不太新，最好是使用Test版

在/etc/portage/package.accept_keywords加入:

sys-kernel/gentoo-sources ~amd64

# emerge -av gentoo-sources
# cd /usr/src/linux
# make localyesconfig
# make -j9
# make modules_install
# make install
# genkernel --install initramfs

# emerge --ask --verbose grub
# mkdir /boot/efi
# grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=Gentoo
# grub-mkconfig -o /boot/grub/grub.cfg

# emerge --ask --verbose sudo
# passwd

无线网卡驱动问题

安装完Gentoo进入系统发现WIFI用不了，以为是interface没有启用

$ ip link
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536 qdisc noqueue state UNKNOWN mode DEFAULT group default qlen 1000
    link/loopback 00:00:00:00:00:00 brd 00:00:00:00:00:00
2: enp4s0f1: <NO-CARRIER,BROADCAST,MULTICAST,UP> mtu 1500 qdisc fq_codel state DOWN mode DEFAULT group default qlen 1000
    link/ether 80:fa:5b:36:93:55 brd ff:ff:ff:ff:ff:ff

没有无线的interface，第一反应以为是内核没启用cfg80211和mac80211,然后看了一下，启用了

$ dmesg | grep iwlwifi

无线驱动没有加载，看最后一行，以为是firmware的问题，然后重装也没啥用，然后查了一下wiki的iwlwifi条目，终于发现了问题

For systems using udev or systemd, it is imperative to configure the kernel to load binary blobs. In this case the wireless card’s firmware is the firmware that needs loaded. More information on configuring the kernel in this manner can be found in the following thread on the Gentoo forums: FW_LOADER_USER_HELPER_FALLBACK.

因为我是使用systemd的，所以需要在内核中配置去加载无线网卡驱动，参考这里讨论 https://www.intel.com/content/www/us/en/support/articles/000005511/network-and-i-o/wireless.html
