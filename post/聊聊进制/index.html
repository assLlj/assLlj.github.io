聊聊进制

进制的表示

首先我们来看看十进制，例如12345，这个数的大小是怎么计算的呢？很简单，我们都知道

12345 = 1×104 + 2×103 + 3×102 + 4×101 + 5×100

那么推广到任意一个十进制数字D：

D = ΣKi×10i

其中Ki是第i位的系数，K $\in$[0, 9], 设D整数位有m位，小数部分有n位，则i$\in$[-n, m-1], 整数部分最高位为m-1, 最低位0,小数部分最高位-1，最低位-n。

当我们推广到一个任意进制（N进制）数字D:

D = ΣKi×Ni

其中N为基数，Ki为第i位的系数，Ni称为第i位的权，i的计数方法如上；

下面是一些例子:

N D 计算方法
2 (101.101)2 (101.101)2 = 1×22+0×21+1×20+1×2-1+0×2-2+1×2-3 = (5.625)10
8 (123.12)8 (123.12)8 = 1×82+2×81+3×80+1×8-1+2×8-2 = (83.15625)10
16 (abc.1)16 (abc.1)16 = 10×162+11×161+12×160 = (2748.0625)10

进制的转换

其他进制–>十进制

如上，略过。

十进制–>二进制

十进制转换到二进制的方法在高中选修就学过，学过C语言的时候也会学习，简而言之就是除2取余,逆序排列,不过这是整数的转换方法，小数部分是不同的。所以十进制转二进制，需要先将十进制数的整数部分和小数部分分别转换后，再加以合并。

整数部分

具体做法是：用2整除十进制整数，可以得到一个商和余数；再用2去除商，又会得到一个商和余数，如此进行，直到商为0时为止，然后把先得到的余数作为二进制数的低位有效位，后得到的余数作为二进制数的高位有效位，依次排列起来。

原理：

设一个十进制整数为(S)10，对应的二进制整数为(则KnKn-1…K0)2,：

(S)10 = (Kn2n + Kn-12n-1 + … + K121 + K020)
    =2(Kn2n-1 + Kn-12n-2 + … + K1)2 + K0

我们能看到，当我们将(S)10除以2，则商为Kn2n-1+Kn-12n-2+…+K1，余数为K0。 所以我们继续下去，就能不断的得到余数为K1，…，Kn-1，Kn,然后我们就能得到二进制数(KnKn-1…K0)2。

小数部分

小数部分采用的方法是乘2取整，顺序排列
具体做法是：用2乘十进制小数，可以得到积，将积的整数部分取出，再用2乘余下的小数部分，又得到一个积，再将积的整数部分取出，如此进行，直到积中的小数部分为零，此时0或1为二进制的最后一位。或者达到所要求的精度为止。

例如:

0.625 = (0.101)2
0.625*2 = 1.25======取出整数部分1
0.25*2 = 0.5========取出整数部分0
0.5*2 = 1==========取出整数部分1

原理:
设一个十进制纯小数为(S)10, 对应的二进制小数为(KnKn-1…K0)2, 即

(S)10 = (Kn2-1 + Kn-12-2 + … + K12-(n-1) + K02-n)
    = 2-1(Kn + Kn-12-1 + … + K12-n + K02-n+1)
2*(S)10 = Kn + 2-1(Kn-1 + Kn-22-1 + … + K02-n)

如上，我们讲(S)10乘2，Kn变成了整数部分，所以我们取整数部分刚好取到了Kn，依次下去，便得到结果。

值得注意的是，二进制表示小数大部分时候是不能有穷表示的，比如0.1用二进制表示就是0.000110011001100110011001100110011.....无限循环下去，因此就有了浮点数的精确度问题。
举个例子，请凭第一感觉说出下面程序的输出是什么

#include<stdio.h>
int main() {
    double i = 0;
    for ( ; i != 10.0; i += 0.1) {
        printf("%f\n", i);
    return 0;
    }
}

其实上面程序会一直跑下去，原因就是计算机无法精确表示0.1，所以i不会刚好等于10.0。 因此，在我们需要比较浮点数是否相等的时候，应该用下面的方式

#include<stdio.h>
#include<math.h>
int main() {
    double i = 0;
    for ( ; fabs(i - 10) > 0.00000001; i += 0.1) {
        printf("%f\n", i);
    }
    return 0;
}
